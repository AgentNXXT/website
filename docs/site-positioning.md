# Agent Site Positioning

Agent Site is the public website, documentation, and marketing surface for Agent Platform.

## Responsibility

Agent Site owns public-facing communication:

- landing pages
- product positioning
- documentation site
- use-case pages
- changelog and release notes
- blog/content
- pricing page later
- customer-facing trust/security pages
- lead capture pages

## Boundary

```text
Agent-Site
  → public website and marketing/docs surface

Agent-Dashboard
  → authenticated product/admin/control dashboard

Agent-Platform
  → product assembly and deployment boundary

Agent-Knowledge
  → source-to-artifact enterprise intelligence product domain
```

## Core Positioning

Agent Platform helps teams turn real-world source material into trusted, evaluated business artifacts and enterprise intelligence workflows.

## Use-Case Pages

Initial pages should include:

- Enterprise Search
- Enterprise Intelligence
- Sales Deck Generation
- Pitch Deck Generation
- Product Documentation
- RFP Response Automation
- Vendor Comparison
- Vendor Proposal Comparison
- Candidate Profile Screening
- Framework Evaluation
- Source-to-Artifact Workflows

## Messaging Rule

Do not sell generic RAG.

Sell concrete business outcomes:

```text
source material → trusted artifact → evaluated decision support
```

## Final Rule

Agent Site explains value. Agent Dashboard controls execution. Agent Platform delivers the product.
